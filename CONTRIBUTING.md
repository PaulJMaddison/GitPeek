# Contributing to RepoVista

Thanks for your interest in contributing 🎉

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes and keep commits focused.
4. Run local checks before opening a PR:

```bash
./gradlew test
./gradlew assembleDebug
```

## Pull Request Guidelines

- Use a clear title and summary of changes.
- Link related issues (e.g., `Closes #123`).
- Add screenshots for UI changes when relevant.
- Keep PRs small and reviewable.

## Code Style

- Follow Kotlin style conventions.
- Keep modules decoupled and feature-scoped.
- Prefer readable, testable code over clever code.

## Reporting Issues

Please use the issue templates for bug reports and feature requests.
